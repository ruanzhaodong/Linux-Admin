阿东的知识库

1. rm - 删除文件/目录

  • 核心功能：
    • 删除指定的文件或目录，操作不可逆，执行前需仔细确认目标。
  • 关键选项：
    • -r（--recursive）：递归删除目录及其下所有文件、子目录（删除目录必须加此选项）；
    • -f（--force）：强制删除，不提示确认（即使文件为只读权限，或目标不存在也不报错）；
    • -i（--interactive）：交互式删除，删除每个文件/目录前都会提示确认（输入 y 确认，n 取消）；
    • -v（--verbose）：显示删除过程的详细信息（如“removed 'test.txt'”）；
    • 常用组合：rm -rf 目录名（强制删除目录及内容，高危操作）、rm -i *.log（交互式删除所有 .log 文件）。
  • 风险提示与补充：
    • 绝对禁止在根目录（/）执行 rm -rf * 或 rm -rf /，会直接删除系统所有文件导致崩溃；
    • 删除隐藏文件时，需明确指定文件名或使用通配符，如 rm -f .bash_history（删除家目录下的历史记录文件）。

2. rmdir - 删除空目录

  • 核心功能：
    • 仅用于删除空目录，无法删除文件或包含文件/子目录的非空目录（与 rm 相比更安全，但功能单一）。
  • 关键选项：
    • -p（--parents）：递归删除空目录及空的父目录，需从最深层目录开始指定。例如 rmdir -p a/b/c，若 c、b、a 均为空，则会依次删除 c、b、a；
    • -v（--verbose）：显示删除目录的详细过程（如“rmdir: removing directory, 'a/b/c'”）。
  • 补充说明与局限性：
    • 若目录非空，执行 rmdir 会报错（如“rmdir: failed to remove 'test': Directory not empty”），此时需先用 rm -rf 目录名 删除目录内容后再删除目录，或直接用 rm -rf 一次性删除；
    • 优势：因仅能删除空目录，可避免误删目录内文件的风险，适合对安全性要求较高的场景；
    • 示例：rmdir empty_dir（删除空目录 empty_dir）、rmdir -p docs/notes（若 notes 和 docs 均为空，则同时删除两者）。
