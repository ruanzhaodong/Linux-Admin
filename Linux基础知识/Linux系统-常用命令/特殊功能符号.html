阿东的知识库

1. > - 输出重定向（覆盖模式）

  • 核心功能：
    • 将命令的输出结果写入指定文件，而非显示在终端；
    • 若目标文件已存在，会覆盖原有内容（谨慎使用）；
    • 若目标文件不存在，会自动创建该文件。
  • 用法示例：
    • echo "Hello World" > test.txt：将字符串写入test.txt（覆盖原有内容）；
    • ls -l > file_list.txt：将目录列表输出保存到file_list.txt；
    • grep "error" app.log > error.log：将匹配"error"的行保存到error.log。
  • 注意事项：
    • 无法重定向标准错误输出（需用2>，见下文补充）；
    • 覆盖操作不可逆，重要文件建议先备份或使用>>追加。

2. >> - 输出重定向（追加模式）

  • 核心功能：
    • 将命令的输出结果追加到指定文件末尾，不影响原有内容；
    • 若目标文件不存在，会自动创建该文件（与>行为一致）。
  • 用法示例：
    • echo "新内容" >> test.txt：在test.txt末尾添加一行；
    • date >> log.txt：将当前时间追加到日志文件；
    • ls -l >> file_list.txt：将目录列表追加到已有文件列表。
  • 典型场景：
    • 日志记录（持续向同一个文件添加内容）；
    • 多步骤输出合并（将多个命令结果汇总到一个文件）。

3. | - 管道符

  • 核心功能：
    • 将前一个命令的标准输出作为后一个命令的标准输入；
    • 实现多命令协作，处理复杂任务（无需临时文件中转）。
  • 用法示例：
    • ls -l | grep ".txt"：列出目录内容并筛选出含".txt"的行；
    • ps aux | grep "nginx"：查找运行中的nginx进程；
    • cat large_file.txt | less：分页查看大文件内容；
    • echo "hello world" | wc -w：统计字符串的单词数量（结果为2）。
  • 高级组合：
    • 多管道串联：ps aux | grep "python" | wc -l（统计python进程数量）；
    • 与重定向结合：ls -l | grep ".log" > log_files.txt（筛选结果保存到文件）。

4. * - 通配符（任意字符匹配）

  • 核心功能：
    • 匹配0个或多个任意字符（包括数字、字母、符号等）；
    • 用于批量选择文件或目录，简化命令输入。
  • 用法示例：
    • ls *.txt：列出所有以".txt"结尾的文件；
    • rm error_*.log：删除所有以"error_"开头且以".log"结尾的文件；
    • cp /data/* ./backup：复制/data目录下的所有内容到backup目录；
    • grep "error" *.log：在所有.log文件中搜索"error"。
  • 注意事项：
    • 默认不匹配以"."开头的隐藏文件（需用.*或结合-a选项）；
    • 匹配范围过宽可能导致意外操作（如rm *会删除当前目录所有可见文件）。

5. ? - 通配符（单个字符匹配）

  • 核心功能：
    • 匹配恰好1个任意字符（长度固定为1）；
    • 用于文件名长度固定但部分字符不同的场景。
  • 用法示例：
    • ls file?.txt：匹配file1.txt、fileA.txt（中间1个字符）；
    • rm image_?.png：删除image_1.png、image_x.png（下划线后1个字符）；
    • ls ???.log：匹配文件名长度为3的.log文件（如abc.log、123.log）。
  • 与*的区别：
    • ? 必须匹配 exactly 1个字符（不能多也不能少）；
    • * 可以匹配0个或多个字符（长度可变）。

6. ~ - 家目录符号

  • 核心功能：
    • 代表当前用户的家目录路径（等价于环境变量$HOME）；
    • 简化路径输入，避免手动输入完整家目录路径（如/home/username）。
  • 用法示例：
    • cd ~：切换到当前用户的家目录；
    • ls ~/Documents：列出家目录下的Documents文件夹内容；
    • cp file.txt ~/Downloads：复制文件到下载目录；
    • ~otheruser：代表指定用户的家目录（如~root指向/root）。
  • 注意事项：
    • 在脚本中使用时，~ 不会被自动解析为家目录，需用$HOME替代；
    • 家目录路径可通过echo ~或echo $HOME查看。

7. .. 和 . - 路径指代符号

  • 核心功能：
    • .：代表当前工作目录；
    • ..：代表上一级目录（父目录）。
  • 用法示例：
    • cd ..：从当前目录切换到上一级目录；
    • ls ./docs：列出当前目录下的docs文件夹内容（./可省略）；
    • cp ../data/file.txt .：复制上一级目录中data文件夹的file.txt到当前目录；
    • cd ../../：连续向上两级目录（父目录的父目录）。
  • 典型场景：
    • 相对路径导航（避免输入完整绝对路径）；
    • 脚本中引用当前目录下的文件（如./script.sh执行当前目录脚本）。

8. && 和 || - 命令逻辑符

  • 核心功能：
    • &&：逻辑"与"，前一个命令执行成功（返回0），才执行后一个命令；
    • ||：逻辑"或"，前一个命令执行失败（返回非0），才执行后一个命令。
  • 用法示例：
    • mkdir test && cd test：创建test目录并在成功后进入该目录；
    • cp file.txt backup/ || echo "复制失败"：复制失败时输出提示；
    • command && echo "成功" || echo "失败"：命令执行结果判断（类似if-else）。
  • 注意事项：
    • 命令执行成功的返回值为0，失败为非0（可通过echo $?查看上一个命令的返回值）；
    • 可串联多个逻辑符：A && B || C && D（按从左到右顺序执行）。

9. $ - 变量引用符

  • 核心功能：
    • 用于引用环境变量或自定义变量的值；
    • 在脚本和命令行中获取变量内容的必备符号。
  • 用法示例：
    • echo $HOME：输出当前用户的家目录路径；
    • echo $PATH：查看系统环境变量PATH（命令搜索路径）；
    • name="Linux" && echo $name：定义变量name并输出其值；
    • echo $?：获取上一个命令的执行结果返回值（0为成功）。
  • 高级用法：
    • ${变量名}：明确变量边界，如echo ${name}file（避免与后续字符混淆）；
    • $1、$2...：脚本中引用第1、第2个参数（位置参数）。
