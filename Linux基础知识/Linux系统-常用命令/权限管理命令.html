阿东的知识库

1. chmod - 修改文件/目录权限

  • 核心功能：
    • 修改文件或目录对不同用户（u/g/o/a）的r/w/x权限；
    • 支持符号模式（直观）和数字模式（简洁）两种操作方式。
  • 符号模式语法：
    • chmod [用户][操作符][权限] 文件名
    • 用户：u（所有者）、g（组）、o（其他）、a（所有）；
    • 操作符：+（添加权限）、-（移除权限）、=（设置固定权限）；
    • 权限：r、w、x。
  • 符号模式示例：
    • chmod u+x file.sh：给所有者添加执行权限；
    • chmod g-w,o-r file.txt：移除组的写权限和其他用户的读权限；
    • chmod a=rwx dir：给所有用户设置读、写、执行权限；
    • chmod ug+w data.csv：给所有者和组添加写权限。
  • 数字模式语法：
    • chmod [三位数字] 文件名（每位数字对应u/g/o的权限总和）；
    • 常用数字组合：777（所有权限）、755（u=rwx,g=rx,o=rx）、644（u=rw,g=r,o=r）。
  • 数字模式示例：
    • chmod 755 script.sh：所有者可执行，组和其他用户可读可执行；
    • chmod 600 secret.txt：仅所有者可读写，其他用户无权限；
    • chmod 700 private_dir：仅所有者可进入并操作目录。
  • 递归修改：
    • 选项 -R：递归修改目录及所有子文件/子目录权限；
    • 示例：chmod -R 755 /data/project（批量设置项目目录权限）。

2. chown 与 chgrp - 修改所有者和所属组

  • chown - 修改文件所有者（及所属组）：
    • 语法：chown [新所有者][:新组] 文件名；
    • 示例：
      • chown alice file.txt：将file.txt的所有者改为alice；
      • chown alice:dev team.doc：将所有者改为alice，所属组改为dev；
      • chown -R bob /data/bob_dir：递归修改目录所有者为bob。
    • 注意：只有root用户可修改文件所有者，普通用户只能修改自己拥有的文件的所属组（且需是该组成员）。
  • chgrp - 修改文件所属组：
    • 语法：chgrp [新组] 文件名；
    • 示例：
      • chgrp dev report.pdf：将report.pdf的所属组改为dev；
      • chgrp -R admin /etc/config：递归修改目录所属组为admin。
    • 替代方案：chown :组名 文件名 与 chgrp 组名 文件名 功能相同。
