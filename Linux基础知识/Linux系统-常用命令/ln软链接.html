1. ln -s 软链接（符号链接）

  • 核心功能：
    • 创建一个特殊的文件（符号链接），其内容是指向另一个文件或目录的路径；
    • 类似Windows系统的"快捷方式"，但功能更基础（不包含额外属性）；
    • 可跨文件系统（分区）创建，支持链接目录。
  • 创建语法：
    • ln -s [源文件/目录路径] [软链接名称]
    • 源文件路径建议使用绝对路径（避免移动软链接后失效）；
    • 示例：ln -s /home/user/docs/report.pdf report_link（创建指向report.pdf的软链接）。
  • 关键特点：
    • 软链接是独立文件，有自己的inode和权限（通常为lrwxrwxrwx）；
    • ls命令中显示为青色文字，且末尾带箭头（如 report_link -> /home/user/docs/report.pdf）；
    • 删除源文件后，软链接会变为"悬空链接"（broken link），访问时提示"没有那个文件或目录"；
    • 修改软链接文件等同于修改源文件（实际操作的是源文件内容）；
    • 可链接不存在的目标（预创建链接，后续创建源文件）。
  • 适用场景：
    • 简化长路径访问（如将深层目录链接到用户家目录）；
    • 软件版本管理（如 ln -s python3.9 python 统一命令入口）；
    • 跨分区共享文件（硬链接不支持跨文件系统）；
    • 临时需要访问的文件/目录，避免频繁切换路径。
  • 管理操作：
    • 查看链接指向：ls -l 软链接名称（显示箭头指向的源路径）；
    • 删除软链接：rm 软链接名称（注意不要加/，否则可能删除源文件内容）；
    • 修改软链接指向：先删除旧链接，再创建新链接（无直接修改命令）。
  • 注意事项：
    • 使用相对路径创建软链接时，相对的是软链接所在目录，而非当前工作目录；
    • 链接目录时，源目录后加不加/效果不同（加/表示链接目录内容，不加表示链接目录本身）；
    • 悬空链接可通过 find / -type l -xtype l 查找（搜索系统中所有无效软链接）。
