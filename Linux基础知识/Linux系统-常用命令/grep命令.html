1. grep - 文本内容搜索工具

  • 核心功能：
    • 在文件或输入流中查找匹配指定模式（字符串或正则表达式）的文本行；
    • 支持按行匹配，输出包含目标模式的完整行；
    • 可用于日志分析、配置文件检索、代码搜索等场景。
  • 基本语法：
    • grep [选项] "模式" [文件...]：在指定文件中搜索模式；
    • 命令 | grep [选项] "模式"：通过管道处理其他命令的输出结果。
  • 关键选项：
    • 匹配控制：
      • -i：忽略大小写（如匹配"Hello"和"hello"）；
      • -w：仅匹配完整单词（避免子字符串匹配，如搜索"cat"不会匹配"category"）；
      • -x：仅匹配整行完全相同的内容；
      • -F：将模式视为固定字符串（不解析正则表达式）。
    • 输出控制：
      • -n：显示匹配行的行号；
      • -H：显示匹配行所在的文件名（多文件搜索时默认显示，单文件时需指定）；
      • -h：抑制文件名显示（多文件搜索时不显示文件名）；
      • -v：反转匹配，输出不包含模式的行；
      • -o：仅输出匹配模式的部分（而非整行）；
      • -c：统计匹配的行数（不显示具体内容）。
    • 递归与范围：
      • -r/-R：递归搜索目录下的所有文件；
      • -l：仅显示包含匹配内容的文件名（不显示具体行）；
      • --include=PATTERN：递归搜索时仅包含符合模式的文件（如 --include="*.txt"）；
      • --exclude=PATTERN：递归搜索时排除符合模式的文件（如 --exclude="*.log"）。
    • 正则表达式：
      • -E：启用扩展正则表达式（支持 |、()、{} 等元字符，无需转义）；
      • -e 模式：指定多个搜索模式（可多次使用，实现逻辑或）。
  • 实用示例：
    • 1. 在文件中搜索字符串：grep "error" app.log（查找app.log中含"error"的行）；
    • 2. 忽略大小写搜索：grep -i "warning" system.log（匹配"WARNING"、"Warning"等）；
    • 3. 显示行号和文件名：grep -nH "timeout" *.conf（在所有.conf文件中搜索并显示行号）；
    • 4. 递归搜索目录：grep -r "import" ./src/（在src目录及子目录的所有文件中找"import"）；
    • 5. 反转匹配（排除指定内容）：grep -v "^#" config.ini（显示config.ini中不是注释的行，^#表示以#开头）；
    • 6. 统计匹配行数：grep -c "failed" access.log（统计access.log中含"failed"的行数）；
    • 7. 结合正则表达式：grep -E "error|warning" syslog（查找含error或warning的行，-E启用扩展正则）。
  • 高级用法：
    • 管道组合：ps aux | grep "nginx"（查找运行中的nginx进程）；
    • 多模式搜索：grep -e "start" -e "end" log.txt（查找含start或end的行）；
    • 搜索二进制文件：grep -a "text" binaryfile（将二进制文件视为文本搜索）；
    • 显示匹配行前后内容：
      • -A 数字：显示匹配行及之后的指定行数（如 -A 3 显示匹配行+后3行）；
      • -B 数字：显示匹配行及之前的指定行数；
      • -C 数字：显示匹配行及前后的指定行数（如 -C 2 显示前后各2行）。
  • 注意事项：
    • 模式中包含特殊字符（如*、.、$）时，需用单引号包裹或转义（如 grep 'file.txt$' 查找以file.txt结尾的行）；
    • 递归搜索大目录时，建议结合--include/--exclude过滤文件类型，提高效率；
    • 与find命令结合使用：find ./ -name "*.php" -exec grep "function" {} \;（在所有php文件中搜索function）。
